# 三元模型的平滑估计

极大似然估计会导致稀疏数据的问题，即使训练集非常大，很多 c(u,v,w)c(u,v,w)c(u,v,w) 和 c(u,v)c(u,v)c(u,v) 也会很小，甚至为 0。因此本节将讨论用于缓和数据稀疏问题的平滑估计（Smoothed Estimation），它的核心思想依赖于 lower-order statistical estimates，即用一元和二元模型的估计去平滑三元模型的估计。本节将讨论两种常用方法：线性插值（Linear Interpolation）和 Discounting Methods，以及与装桶（Bucketing）结合的线性插值。

# 线性插值

定义三元、二元和一元模型的极大似然估计为：

qML(w∣u,v)=c(u,v,w)c(u,v)q_{ML}(w|u,v)=\frac{c(u,v,w)}{c(u,v)} qML​(w∣u,v)=c(u,v)c(u,v,w)​

qML(w∣v)=c(u,v,w)c(v)q_{ML}(w|v)=\frac{c(u,v,w)}{c(v)} qML​(w∣v)=c(v)c(u,v,w)​

qML(w)=c(u,v,w)c()q_{ML}(w)=\frac{c(u,v,w)}{c()} qML​(w)=c()c(u,v,w)​

c(w)c(w)c(w) 表示单词 www 在训练集中出现的次数，c(⋅)c(\cdot)c(⋅) 表示训练集中所有单词的数量。

它们有各自的优缺点：一元模型估计出的参数的分子和分母一定会大于 0（因为每个词在训练集中都至少出现了一次），但它完全忽视了上下文信息。而三元模型考虑了上下文信息，但很多参数会被估计为 0，数据更稀疏。二元模型则介于两者之间。

因此线性插值（Linear Interpolation）的思想是把三种估计加权平均，定义三元估计为：

q(w∣u,v)=λ1×qML(w∣u,v)+λ2×qML(w∣v)+λ3×qML(w)q(w|u,v)=\lambda_1 \times q_{ML}(w|u,v) + \lambda_2 \times q_{ML}(w|v) + \lambda_3 \times q_{ML}(w) q(w∣u,v)=λ1​×qML​(w∣u,v)+λ2​×qML​(w∣v)+λ3​×qML​(w)

其中 λ1≥0,λ2≥0,λ3≥0\lambda_1 \geq 0, \lambda_2 \geq 0, \lambda_3 \geq 0λ1​≥0,λ2​≥0,λ3​≥0，且 λ1+λ2+λ3=1\lambda_1 + \lambda_2 + \lambda_3 = 1λ1​+λ2​+λ3​=1。

估计 λ\lambdaλ 的值有多种方式，一种常见的方法是：假设我们有一些不同于训练集和测试集的额外的数据，称之为验证集（development set/validation set）。c′(u,v,w)c'(u, v, w)c′(u,v,w) 为三元组 (u,v,w)(u, v, w)(u,v,w) 在验证集中出现的次数。则验证集上的 log-likelihood（对数似然函数值）为：

L(λ1,λ2,λ3)=∑u,v,wc′(u,v,w)log⁡q(w∣u,v)L(\lambda_1,\lambda_2,\lambda_3)=\sum_{u,v,w}c'(u,v,w) \log q(w|u,v) L(λ1​,λ2​,λ3​)=u,v,w∑​c′(u,v,w)logq(w∣u,v)

我们要选择使 L(λ1,λ2,λ3)L(\lambda_1,\lambda_2,\lambda_3)L(λ1​,λ2​,λ3​) 尽可能大，且满足 λ1≥0,λ2≥0,λ3≥0\lambda_1 \geq 0, \lambda_2 \geq 0, \lambda_3 \geq 0λ1​≥0,λ2​≥0,λ3​≥0，λ1+λ2+λ3=1\lambda_1 + \lambda_2 + \lambda_3 = 1λ1​+λ2​+λ3​=1 的 λ\lambdaλ 值：

综上，我们引入了三个平滑参数 λ1,λ2,λ3\lambda_1,\lambda_2,\lambda_3λ1​,λ2​,λ3​，这三个参数可以被理解为是三元、二元、一元极大似然估计的置信度或权重。比如，如果 λ1\lambda_1λ1​ 的值非常接近于 1 ，则意味着我们认为 qML(w∣u,v)q_{ML}(w|u,v)qML​(w∣u,v) 的意义很大；相反，如果 λ1\lambda_1λ1​ 接近于 0，则意味我们认为 qML(w∣u,v)q_{ML}(w|u,v)qML​(w∣u,v) 的意义不大。

在实际操作中，我们应该让 λ1,λ2,λ3\lambda_1,\lambda_2,\lambda_3λ1​,λ2​,λ3​ 根据不同的二元组 (u,v)(u, v)(u,v) 进行改变。具体来说，当 c(u,v)c(u, v)c(u,v) 更大时，我们应该将 λ1\lambda_1λ1​ 变得更大。因为我们认为当 c(u,v)c(u, v)c(u,v) 较大时，应该给三元估计更高的置信度。

需要保证当 c(u,v)=0c(u, v) = 0c(u,v)=0 时，λ1=0\lambda_1 =0λ1​=0。因为此时 qML(w∣u,v)=c(u,v,w)c(u,v)q_{ML}(w|u,v)=\frac{c(u,v,w)}{c(u,v)}qML​(w∣u,v)=c(u,v)c(u,v,w)​ 的分母为 0，三元极大似然估计的无法定义。同样，如果 c(u,v)c(u, v)c(u,v) 和 c(v)c(v)c(v) 都为 0，那么需要保证 $ \lambda_1 = \lambda_2 = 0$，因为此时三元和二元极大似然估计都无法定义。

另一种更简单的定义 λ\lambdaλ 的方法是：

λ1=c(u,v)c(u,v)+γ\lambda_1 = \frac{c(u,v)}{c(u,v)+ \gamma} λ1​=c(u,v)+γc(u,v)​

γ>0\gamma > 0γ>0 是该方法中唯一的参数。这种方法可以保证 λ1≥0,λ2≥0,λ3≥0\lambda_1 \geq 0, \lambda_2 \geq 0, \lambda_3 \geq 0λ1​≥0,λ2​≥0,λ3​≥0 和 λ1+λ2+λ3=1\lambda_1 + \lambda_2 + \lambda_3 = 1λ1​+λ2​+λ3​=1。

λ1\lambda_1λ1​ 会随着 c(u,v)c(u, v)c(u,v) 的增大而增大，λ2\lambda_2λ2​ 会随着 c(v)c(v)c(v) 的增大而增大。如果 c(u,v)=0c(u, v) = 0c(u,v)=0，则有 λ1=0\lambda_1 = 0λ1​=0；如果 c(v)=0c(v) = 0c(v)=0，则有 λ2=0\lambda_2 = 0λ2​=0。γ\gammaγ 依然取能最大化验证集上的 log-likelihood 的值。

这种决定参数的方式非常粗糙，而且一般情况下并不是最优的。但它足够简单，且有较好的实际应用效果。

# 线性插值 + bucketing
